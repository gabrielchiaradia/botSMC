01 Conectarse al servidor (SSH)

Desde Windows (CMD o PowerShell)

ssh -i C:\Users\Gabriel\.ssh\id_ed25519 [email protected]

Cambiar al usuario botuser

El bot corre como botuser, no como root. Después de conectarte:

su - botuser
cd ~/smc-bot

Desconectarse

exit           # salir de botuser → vuelve a root
exit           # salir del VPS → vuelve a tu PC
02 Bot: iniciar, parar y reiniciar

El bot corre dentro de un container Docker. Todos los comandos se ejecutan desde ~/smc-bot como botuser.

Iniciar el bot

docker compose up -d --build smc-bot watchtower

El flag -d lo ejecuta en background. El bot sigue corriendo aunque cierres la sesión SSH.

Parar el bot

docker compose down

Reiniciar el bot (sin reconstruir)

docker compose restart smc-bot

Reconstruir y reiniciar (después de cambios de código)

docker compose down
docker compose build --no-cache smc-bot
docker compose up -d smc-bot watchtower

Ver si el bot está corriendo

docker compose ps
Inicio automático El bot tiene restart: unless-stopped en docker-compose.yml. Si el VPS reinicia, el bot se levanta solo.
03 Ver logs del bot

Logs en tiempo real (Ctrl+C para salir)

docker compose logs -f smc-bot

Últimas N líneas

docker compose logs --tail 50 smc-bot

Log del archivo (persiste entre reinicios)

tail -50 ~/smc-bot/logs/smc_bot.log

Seguir el archivo en vivo

tail -f ~/smc-bot/logs/smc_bot.log

Ver uso de recursos

docker stats smc-bot
04 Dashboard web: iniciar y parar

El dashboard es un container nginx separado que sirve el HTML en el puerto 8080.

Iniciar el dashboard (como root)

docker run -d \
  --name smc-dashboard \
  --restart unless-stopped \
  -p 8080:80 \
  -v /home/botuser/smc-bot/dashboard/index.html:/usr/share/nginx/html/index.html:ro \
  -v /home/botuser/smc-bot/dashboard/nginx.conf:/etc/nginx/conf.d/default.conf:ro \
  -v /home/botuser/smc-bot/logs:/usr/share/nginx/html/logs:ro \
  -v /home/botuser/smc-bot/backtest/results:/usr/share/nginx/html/backtest:ro \
  nginx:latest

Acceder al dashboard

Abrí en tu navegador: http://167.179.114.168:8080

Parar el dashboard

docker rm -f smc-dashboard

Reiniciar el dashboard (después de cambios)

docker rm -f smc-dashboard
docker run -d \
  --name smc-dashboard \
  --restart unless-stopped \
  -p 8080:80 \
  -v /home/botuser/smc-bot/dashboard/index.html:/usr/share/nginx/html/index.html:ro \
  -v /home/botuser/smc-bot/dashboard/nginx.conf:/etc/nginx/conf.d/default.conf:ro \
  -v /home/botuser/smc-bot/logs:/usr/share/nginx/html/logs:ro \
  -v /home/botuser/smc-bot/backtest/results:/usr/share/nginx/html/backtest:ro \
  nginx:latest
05 Subir cambios al servidor (Git)

Paso 1: Desde tu PC Windows, pushear a GitHub

cd C:\Users\Gabriel\Downloads\smc-bot-improved
git add .
git commit -m "descripción del cambio"
git push

Paso 2: Desde el VPS, bajar los cambios

cd ~/smc-bot
git pull

Paso 3: Reconstruir el bot con los cambios

docker compose down
docker compose build --no-cache smc-bot
docker compose up -d smc-bot watchtower
Importante El .env no se sube a Git (está en .gitignore). Si cambiás el .env en tu PC, tenés que editarlo manualmente en el VPS también.

07 Correr backtests en el VPS

Los backtests se ejecutan con Python directamente, sin Docker.

Backtest básico

cd ~/smc-bot
python3 scripts/run_backtest.py --symbol ETHUSDT --timeframe 1h --dias 90

Con perfil y trailing

python3 scripts/run_backtest.py --symbol ETHUSDT --timeframe 1h --dias 90 --strategy ob_bos --trailing hibrido

Comparar trailing

python3 scripts/run_backtest.py --symbol ETHUSDT --timeframe 1h --dias 90 --compare-trailing

Con múltiples trades

python3 scripts/run_backtest.py --symbol ETHUSDT --timeframe 1h --dias 360 --max-trades 2

Los resultados se guardan en backtest/results/ y se ven en el dashboard.

08 Correr análisis sistemático

El script run_systematic.py prueba automáticamente múltiples combinaciones y encuentra la configuración óptima.

Análisis completo (todos los pares y timeframes)

python3 scripts/run_systematic.py --dias 180

Forzar par y timeframe (salta Fase 1)

python3 scripts/run_systematic.py --symbol ETHUSDT --timeframe 1h --dias 360

Con múltiples trades

python3 scripts/run_systematic.py --symbol ETHUSDT --timeframe 1h --dias 360 --max-trades 2

Fases del análisis

FaseQué prueba
1Mejor par + timeframe (BTC/ETH × 15m/1h)
2Mejor perfil + trailing (base/ob_bos × none/escalones/hibrido)
3Mejor score mínimo + ATR multiplier
4Mejor TP/RR ratio (1:1.5 a 1:5.0)
5Re-test trailing con el RR óptimo
09 Pasar de Testnet a Real
Precaución Solo pasar a real después de verificar que el bot funciona correctamente en testnet durante al menos una semana. El trading con futuros y apalancamiento conlleva riesgo de pérdida total del capital.

Pasos

  1. Crear API keys en Binance real
  2. Editar .env en el VPS:
TESTNET=false
BINANCE_API_KEY=tu_key_real
BINANCE_API_SECRET=tu_secret_real
  1. Reconstruir el bot:
docker compose down
docker compose up -d smc-bot watchtower
Recomendación Empezá con capital mínimo ($50-100) y RISK_PER_TRADE=0.01 (1%). Verificá que las primeras operaciones se ejecutan correctamente antes de aumentar el capital.
