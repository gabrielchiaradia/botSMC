03 Ventanas horarias (Trading Windows)

Las ventanas definen en qué horas UTC el bot puede abrir trades. Fuera de esas horas, el bot analiza y detecta señales pero no opera.

VentanaHorario UTCGMT-3 (Argentina)Qué cubre
8-1108:00 – 11:0005:00 – 08:00Apertura London
13-1813:00 – 18:0010:00 – 15:00Core New York
7-1607:00 – 16:0004:00 – 13:00London completa
12-2112:00 – 21:0009:00 – 18:00New York completa
2-1002:00 – 10:0023:00 – 07:00Asia completa
8-11,13-18combinadocombinadoAperturas London + NY
(vacío)24h24hSin filtro horario

Formato

Las ventanas se definen como HH-HH separadas por coma. Ejemplo: 8-11,13-18 significa que solo se opera de 8 a 11 y de 13 a 18 UTC.

Tip Una ventana vacía (--windows "") desactiva el filtro y opera 24 horas. Útil para comparar con y sin filtro.

08 Mega-comparación: comandos listos para copiar

Comandos encadenados con && para correr uno tras otro. Cada grupo tarda entre 5–30 minutos según los días.

Batch 1: ETH/1h ventanas horarias (360 días)

python scripts/run_backtest.py --symbol ETHUSDT --timeframe 1h --dias 360 --windows 8-11 && python scripts/run_backtest.py --symbol ETHUSDT --timeframe 1h --dias 360 --windows 13-18 && python scripts/run_backtest.py --symbol ETHUSDT --timeframe 1h --dias 360 --windows 8-11,13-18 && python scripts/run_backtest.py --symbol ETHUSDT --timeframe 1h --dias 360 --windows ""

Batch 2: ETH/15m ventanas horarias (180 días)

python scripts/run_backtest.py --symbol ETHUSDT --timeframe 15m --dias 180 --windows 8-11 && python scripts/run_backtest.py --symbol ETHUSDT --timeframe 15m --dias 180 --windows 13-18 && python scripts/run_backtest.py --symbol ETHUSDT --timeframe 15m --dias 180 --windows 8-11,13-18 && python scripts/run_backtest.py --symbol ETHUSDT --timeframe 15m --dias 180 --windows ""

Batch 3: Systematic completo ETH/1h con ventanas (360 días)

python scripts/run_systematic.py --symbol ETHUSDT --timeframe 1h --dias 360 --max-trades 2 --windows 8-11,13-18

Batch 4: Comparación 15m ventanas granulares (180 días)

python scripts/run_backtest.py --timeframe 15m --dias 180 --windows 7-9 && python scripts/run_backtest.py --timeframe 15m --dias 180 --windows 9-11 && python scripts/run_backtest.py --timeframe 15m --dias 180 --windows 11-13 && python scripts/run_backtest.py --timeframe 15m --dias 180 --windows 13-15 && python scripts/run_backtest.py --timeframe 15m --dias 180 --windows 15-17 && python scripts/run_backtest.py --timeframe 15m --dias 180 --windows 17-19

Batch 5: RR sweep en la mejor config

python scripts/run_backtest.py --timeframe 1h --dias 360 --windows 8-11,13-18 --max-trades 2 --rr 1.5 && python scripts/run_backtest.py --timeframe 1h --dias 360 --windows 8-11,13-18 --max-trades 2 --rr 2.0 && python scripts/run_backtest.py --timeframe 1h --dias 360 --windows 8-11,13-18 --max-trades 2 --rr 2.5 && python scripts/run_backtest.py --timeframe 1h --dias 360 --windows 8-11,13-18 --max-trades 2 --rr 3.0
09 Cómo interpretar los resultados

Métricas clave (en orden de importancia)

MétricaQué buscarRed flag
Profit Factor≥ 1.3 es bueno, ≥ 1.5 excelente< 1.0 pierde dinero
Win Rate35–55% con RR 1:2 es viable< 25% con cualquier RR
Total trades≥ 50 para ser estadísticamente confiable< 20 trades = no confiar
Max Drawdown≤ 15% es cómodo, ≤ 10% ideal> 30% es peligroso
RetornoRelativo al DD. Retorno/DD > 2 es buenoAlto retorno con alto DD = suerte

Señales de sobreajuste (overfitting)

Si una configuración da resultados espectaculares en 90 días pero mediocres en 360 días, es sobreajuste. Siempre validá con el período más largo posible.

Regla de oro Preferí configuraciones con PF alto y DD bajo sobre configuraciones con retorno alto y DD alto. Un PF de 1.5 con DD 10% es mucho mejor que un PF de 1.2 con DD 30%, aunque el segundo tenga más retorno bruto.

Flujo recomendado de testing

  1. Correr Batch 1 (ventanas en 1h) → encontrar la mejor ventana
  2. Correr Batch 2 (ventanas en 15m) → comparar con 1h
  3. Con la mejor ventana, correr Batch 3 (systematic completo)
  4. Afinar con Batch 4 (ventanas granulares) si 15m es mejor
  5. Confirmar RR con Batch 5
  6. Configurar el bot con los parámetros ganadores
10 Comparar resultados: compare_results.py

Después de correr múltiples backtests, usá este script para comparar todos los resultados en una tabla con colores y ranking automático.

ParámetroValoresDefaultDescripción
filespatrones globtodosArchivos JSON a comparar. Soporta wildcards (bt_ethusdt*.json)
--dirrutabacktest/resultsDirectorio donde buscar JSONs
--top1–100todosMostrar solo los N mejores resultados
--sortret, pf, wr, dd, trades, sharperetMétrica para ordenar
--min-trades1–10000Filtrar resultados con menos de N trades
--csvnombre.csv—Exportar comparación a CSV

Comparar todos los backtests

python scripts/compare_results.py

Top 5 ordenados por Profit Factor

python scripts/compare_results.py --top 5 --sort pf

Solo resultados confiables (50+ trades), menor drawdown primero

python scripts/compare_results.py --min-trades 50 --sort dd

Comparar solo los de ETH/1h

python scripts/compare_results.py bt_ethusdt_1h*.json

Comparar solo los de 15m con ventanas

python scripts/compare_results.py bt_ethusdt_15m*w*.json

Exportar a CSV para Excel

python scripts/compare_results.py --csv comparacion.csv

Flujo completo recomendado

# 1. Correr backtests con diferentes configs
python scripts/run_backtest.py --timeframe 1h --dias 360 --windows 8-11
python scripts/run_backtest.py --timeframe 1h --dias 360 --windows 13-18
python scripts/run_backtest.py --timeframe 1h --dias 360 --windows 8-11,13-18
python scripts/run_backtest.py --timeframe 15m --dias 180 --windows 8-11,13-18

# 2. Comparar todos los resultados
python scripts/compare_results.py --sort pf --min-trades 30

# 3. Ver solo el top 3
python scripts/compare_results.py --top 3 --sort pf
Tip La tabla muestra colores automáticos: verde para métricas buenas, amarillo para mediocres, rojo para malas. PF ≥ 1.3 es verde, WR ≥ 40% es verde, DD ≤ 10% es verde.
